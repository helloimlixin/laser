\begin{abstract}
We study a two-stage generative modeling pipeline in which the first stage learns a sparse dictionary representation and the second stage learns a transformer prior over the resulting sparse codes. The proposed model, LASER, replaces a single nearest-neighbor vector-quantized latent with a small learned set of dictionary atoms and coefficients at each latent site or patch. This gives the autoencoder a compositional bottleneck while retaining a discrete interface for generative modeling through coefficient quantization. Stage 1 uses a VQGAN/DDPM-style convolutional encoder-decoder with bottleneck attention and a straight-through sparse projection. Stage 2 uses a spatial-depth transformer that separates long-range spatial structure from the within-site sparse tuple. We report image experiments on CelebA-HQ and related development runs, including reconstruction, FID, support/coefficient prediction, quantized versus real-valued coefficient ablations, and patch-layout sweeps. We also document an audio extension on VCTK log-spectrograms, including the implemented single-channel data path, audio-domain reconstruction metrics, completed VQ-VAE audio baselines, and partial LASER audio training evidence. The current evidence suggests that sparse dictionary latents can substantially improve image distributional quality relative to a VQ-VAE baseline at similar reconstruction quality, while the audio LASER results require a completed stage-2 run before making final cross-modal claims.
\end{abstract}

\section{Introduction}

Two-stage generative models usually learn an autoencoder first and then train a prior in the learned latent space. Vector-quantized autoencoders make the second stage easy because each latent site is a single discrete token, but they force each site to choose one codebook vector. This paper investigates a different bottleneck: each latent site or patch is reconstructed from a sparse linear combination of learned dictionary atoms. The resulting representation keeps a compact token interface for the prior while allowing each location to combine several reusable parts.

LASER, short for Learnable Adaptive Structured Embedding Representation, is designed around this sparse bottleneck. Stage 1 maps an image or spectrogram to a latent tensor, projects local latent vectors or patches onto a sparse dictionary model, and decodes from the sparse reconstruction. Stage 2 models the sparse support and coefficient representation. For quantized coefficients, both atom ids and coefficient bins become discrete tokens. For real-valued coefficients, atom ids remain discrete and coefficients are modeled with regression or Gaussian heads.

This draft makes three contributions.

\begin{itemize}
    \item We describe a maintained LASER architecture with a VQGAN/DDPM-style stage-1 backbone, normalized sparse dictionary atoms, OMP-style patch coding, coefficient quantization, and a metadata-rich cache interface.
    \item We describe a stage-2 spatial-depth transformer prior that models spatial layout separately from the sparse tuple at each spatial site.
    \item We report the current experimental record in both vision and audio modalities. Vision has completed stage-1, stage-2, and coefficient-mode experiments. Audio has a complete VQ-VAE baseline and a working LASER stage-1 path, but the local artifacts do not yet contain a completed LASER audio token cache or stage-2 run.
\end{itemize}

\section{Model}

\subsection{Stage 1: Sparse Dictionary Autoencoder}

Let $x$ denote an input image or a normalized audio spectrogram. Stage 1 first applies an encoder $E_\theta$ and obtains a dense latent tensor
\begin{equation}
    z_e = E_\theta(x) \in \mathbb{R}^{C \times H \times W}.
\end{equation}
The decoder $G_\psi$ maps a sparse reconstructed latent tensor back to data space. In the maintained configuration, $E_\theta$ and $G_\psi$ use a VQGAN/DDPM-style ResNet backbone: a $3 \times 3$ input convolution, residual blocks at each resolution, optional attention at configured resolutions, a middle ResNet-attention-ResNet block at the coarsest resolution, and a mirrored decoder with nearest-neighbor upsampling followed by convolution. Unless otherwise stated, attention is only used in the middle block.

\paragraph{DDPM-style multiscale tower.}
The stage-1 backbone follows the convolutional U-Net design pattern used by DDPM~\citep{ho2020ddpm} and VQGAN/RQ-VAE models: each resolution level is a stack of residual blocks, spatial resolution changes are learned by explicit downsampling or upsampling operators, and self-attention can be inserted at selected resolutions. In LASER this tower is used as an autoencoder rather than as a diffusion denoiser. There is no timestep embedding and no encoder-decoder skip connection across the sparse bottleneck; all information that reaches the decoder must pass through $z_{\mathrm{dl}}$. Thus the architecture is U-Net-like in its multiscale residual/attention blocks, but not a skip-connected U-Net.

For an input resolution $R$ and channel multiplier schedule $(m_0,\ldots,m_L)$, the encoder has $L$ learned downsampling steps and produces a latent grid of size $R/2^L$. The maintained default for $256 \times 256$ CelebA-HQ runs uses two downsampling steps, producing a $64 \times 64$ latent grid before patch tokenization. Stronger high-resolution sweeps can set an explicit schedule such as $(1,1,2,2,4,4)$, giving a $256 \rightarrow 8 \times 8$ tower with wider low-resolution blocks. Each encoder level first applies residual blocks at the current resolution, then, except for the last level, pads the feature map on the right and bottom and applies a $3 \times 3$ stride-2 convolution. This mirrors the DDPM convention of avoiding pooling-only resolution changes while keeping the downsample operator local and learned.

Each residual block uses GroupNorm, SiLU, a $3 \times 3$ convolution, another GroupNorm-SiLU pair, dropout, and a second $3 \times 3$ convolution, plus an identity or learned $1 \times 1$ shortcut when the channel count changes:
\begin{equation}
    \mathrm{ResBlock}(h)
    =
    S(h) + W_2 \phi\!\left(N_2\!\left(W_1 \phi(N_1(h))\right)\right),
\end{equation}
where $N_i$ are GroupNorm layers, $\phi$ is SiLU, and $S$ is either identity or a channel-matching projection. Attention blocks also follow the DDPM/VQGAN template: the feature map is normalized, projected with $1 \times 1$ convolutions to queries, keys, and values, flattened to a sequence of $HW$ spatial positions, mixed with scaled dot-product attention, projected back with a $1 \times 1$ convolution, and added residually:
\begin{equation}
    \mathrm{Attn}(h)
    =
    h + W_o\!\left(V\,\mathrm{softmax}(QK^\top/\sqrt{C})^\top\right).
\end{equation}
The coarsest encoder feature always passes through a middle block
\begin{equation}
    \mathrm{ResBlock} \rightarrow \mathrm{AttnBlock} \rightarrow \mathrm{ResBlock}
\end{equation}
when \texttt{use\_mid\_attention=true}. Additional attention can be enabled by listing spatial resolutions in \texttt{attn\_resolutions}.

\paragraph{Decoder.}
The decoder reverses the same design. It first maps the sparse latent tensor to the deepest channel width with a $3 \times 3$ convolution, applies the same middle ResBlock-AttnBlock-ResBlock module, then walks back through the channel multiplier schedule from coarse to fine. Each decoder level uses residual blocks at the current resolution; non-final levels upsample by nearest-neighbor interpolation followed by a $3 \times 3$ convolution. The decoder can use one extra residual block per level, matching common VQGAN/RQ-VAE decoder practice where the generative path receives slightly more capacity than the encoder. A final GroupNorm-SiLU-$3 \times 3$ convolution maps the last feature map to RGB for images or one log-spectrogram channel for audio.

For each latent unit $u$, the bottleneck constructs a vector $h_u$. In per-site mode, $h_u \in \mathbb{R}^{C}$ is one latent vector. In patch mode, $h_u \in \mathbb{R}^{C p^2}$ is an unfolded $p \times p$ latent patch. LASER learns a dictionary
\begin{equation}
    D = [d_1,\ldots,d_M] \in \mathbb{R}^{m \times M}, \qquad \|d_j\|_2 = 1,
\end{equation}
and approximates each $h_u$ by solving
\begin{equation}
    a_u^\star \approx \operatorname*{arg\,min}_{a \in \mathbb{R}^{M}}
    \frac{1}{2}\|h_u - D a\|_2^2
    \quad \mathrm{s.t.} \quad
    \|a\|_0 \leq K .
\end{equation}
The implementation uses top-$K$ correlation selection followed by a regularized least-squares solve in per-site mode and batched Orthogonal Matching Pursuit in patch mode. Patch reconstructions are stitched back to the latent grid using center-crop, tiling, or Hann overlap-add. When patch stride equals patch size, tiling is used automatically.

The sparse reconstruction is
\begin{equation}
    z_{\mathrm{dl}} = \mathcal{R}(\{D a_u^\star\}_u),
\end{equation}
where $\mathcal{R}$ denotes the inverse layout operation from sites or patches to the latent tensor. Training uses a straight-through replacement
\begin{equation}
    z_{\mathrm{st}} = z_e + \mathrm{sg}(z_{\mathrm{dl}} - z_e),
\end{equation}
so the decoder receives the sparse latent but the encoder still receives gradient signal through the identity branch.

The stage-1 objective is
\begin{align}
    \mathcal{L}_{\mathrm{s1}}
    &=
    \mathcal{L}_{\mathrm{rec}}(x,\hat{x})
    + \lambda_{\mathrm{bn}}\mathcal{L}_{\mathrm{bn}}
    + \lambda_{\mathrm{perc}}\mathcal{L}_{\mathrm{perc}}
    + \lambda_{\mathrm{coh}}\mathcal{L}_{\mathrm{coh}}, \\
    \mathcal{L}_{\mathrm{bn}}
    &=
    \|z_{\mathrm{dl}} - \mathrm{sg}(z_e)\|_2^2
    + \beta\|\mathrm{sg}(z_{\mathrm{dl}})-z_e\|_2^2 .
\end{align}
For images, $\mathcal{L}_{\mathrm{rec}}$ is pixel MSE in normalized image space and $\mathcal{L}_{\mathrm{perc}}$ is LPIPS when enabled. For audio spectrograms, $\mathcal{L}_{\mathrm{perc}}$ is disabled and the model can add an audio-energy matching term that compares frame-level and global log energy reconstructed from the spectrogram.

\subsection{Token Cache}

The token cache is the explicit contract between stage 1 and stage 2. It stores sparse supports, coefficients, latent shape metadata, patch layout, coefficient quantization settings, and enough stage-1 backbone metadata to decode samples later.

For quantized sparse codes, each sparse slot contributes an atom token and a coefficient-bin token:
\begin{equation}
    y_{u} = [i_{u,1}, b_{u,1}, i_{u,2}, b_{u,2}, \ldots, i_{u,K}, b_{u,K}],
\end{equation}
where $i_{u,k} \in \{0,\ldots,M-1\}$ and $b_{u,k}$ is offset by $M$ in the shared vocabulary. The cache depth is therefore $D_t=2K$. For real-valued sparse codes, the cache stores atom ids in the token tensor and coefficients in a separate floating-point tensor, so $D_t=K$.

The cache writer canonicalizes sparse pairs by sorting supports by atom id. This matters because the sparse approximation is an unordered set, while the transformer consumes an ordered sequence.

\subsection{Stage 2: Spatial-Depth Prior}

The default prior is a spatial-depth transformer. It models a sparse token grid $Y \in \mathbb{Z}^{H_c \times W_c \times D_t}$ in raster spatial order and depth order within each site. Conceptually,
\begin{equation}
    p(Y) =
    \prod_{s=1}^{H_c W_c}
    \prod_{d=1}^{D_t}
    p(y_{s,d} \mid Y_{<s}, y_{s,<d}).
\end{equation}
The implementation decomposes this into two modules. A spatial transformer computes contextual features for spatial sites. A depth transformer predicts the ordered sparse tuple at each site using the spatial context and previous depth tokens. For quantized caches, the depth head predicts the shared atom-plus-coefficient vocabulary. For real-valued caches, it predicts atom logits and either direct coefficient values or Gaussian coefficient parameters.

This design preserves the distinction between where a code occurs and which sparse atom-coefficient pair fills a slot. It also supports patch-based caches where the cache grid differs from the full latent grid; the stage-2 decoder uses cache metadata to reconstruct the latent before calling the stage-1 decoder.

\subsection{Audio Adaptation}

For VCTK audio, the data module converts waveform clips to normalized single-channel log-power spectrograms. Each clip is resampled to 16 kHz, cropped or zero-padded to 32,768 samples, transformed with an STFT using $n_{\mathrm{fft}}=1024$ and hop length 256, log-compressed, resized to $128 \times 128$, and normalized with mean and standard deviation 0.5. Evaluation reconstructs audio-domain quantities from the predicted spectrogram and reports log-spectral distance (LSD), log-magnitude MSE/L1, spectral convergence, log-mel L1, and optional ViSQOL when available. In the local VCTK runs reported here, ViSQOL was not available.

Given reference magnitude $A$ and reconstructed magnitude $\hat{A}$, the reported spectral convergence is
\begin{equation}
    \mathrm{SC}(A,\hat{A}) = \frac{\|A-\hat{A}\|_F}{\|A\|_F},
\end{equation}
and LSD is computed as an RMS distance between log magnitudes over time-frequency bins. For qualitative logging, Griffin-Lim reconstruction is used to invert predicted magnitudes to waveform clips. These audio metrics are only used for audio batches; image metrics such as PSNR and FID are suppressed for VCTK.

\section{Experiments}

\subsection{Experimental Protocol}

Unless otherwise specified, image experiments use CelebA-HQ at $256 \times 256$, a VQGAN-style stage-1 backbone, coefficient bound 8.0, 256 coefficient bins for quantized LASER caches, AdamW/Adam optimization through PyTorch Lightning, and a spatial-depth stage-2 prior with $d_{\mathrm{model}}=512$, 8 attention heads, 6 transformer layers, feed-forward width 2048, and dropout 0.1. Audio experiments use VCTK at $128 \times 128$ spectrogram resolution.

The tables distinguish completed runs from partial or interrupted runs. This is important for audio: the local repository contains completed VQ-VAE audio stage-1 and stage-2 runs, but the available LASER audio runs were interrupted before token-cache extraction and stage-2 training.

\section{Discussion}

The vision results show a clear difference between pixel distortion and distributional quality. VQ-VAE has lower reconstruction loss and higher PSNR than LASER in the completed CelebA-HQ comparison, but LASER has much lower FID. This suggests that the sparse dictionary bottleneck changes the reconstruction errors in a way that is less harmful to distributional statistics, possibly because the latent must be composed from reusable atoms rather than selected from a single nearest code.

The coefficient comparison supports quantized coefficients for the current stage-2 design. Quantization improves reconstruction in the completed $256 \times 256$ sweep and gives the prior a clean discrete likelihood. The real-valued coefficient path remains useful for analysis, but it needs a more explicit probabilistic coefficient model before it should be treated as a final generative likelihood.

The audio path is promising but incomplete. The spectrogram data module, metrics, logging, and single-channel model configs are implemented and tested. The completed VQ-VAE audio baseline provides a reference. LASER audio stage 1 runs and can reach competitive validation LSD early, but the current artifact set does not support a final LASER audio claim because no completed token cache or stage-2 run is available.

\section{Limitations}

The current paper draft has four limitations. First, several stage-2 vision jobs were interrupted by the scheduler and are reported as partial validation evidence, not final test results. Second, final sample quality metrics for generated images are not yet reported. Third, the audio LASER experiment is not complete. Fourth, the theoretical sparse-recovery story is intentionally modest: the implementation uses normalized dictionaries and projected gradients, but it does not enforce coherence conditions strong enough for classical OMP recovery guarantees at the maintained defaults.

\section{Related Work}

LASER is related to vector-quantized autoencoders such as VQ-VAE~\citep{oord2017vqvae} and VQGAN~\citep{esser2021vqgan}, which provide discrete latent spaces for autoregressive priors. It differs by replacing one-code-per-site quantization with a sparse dictionary model. The sparse bottleneck is connected to dictionary learning and sparse coding~\citep{olshausen1997sparse,aharon2006ksvd,tropp2007omp}, but it is trained as part of a deep straight-through autoencoder rather than as a standalone sparse coding objective. The second stage follows the broad recipe of latent generative modeling with transformers~\citep{vaswani2017attention}, but uses a spatial-depth factorization tailored to sparse tuples rather than a flat pixel or token sequence. The audio experiments connect to neural audio compression and spectrogram autoencoding, where reconstruction quality is often assessed with log-spectral and perceptual metrics rather than image metrics.

\section{Reproducibility}

The maintained code paths are \texttt{train_stage1_autoencoder.py} for stage-1 autoencoder training, \texttt{cache.py} for token-cache extraction, \texttt{train_stage2_prior.py} for stage-2 prior training, and \texttt{sample.py} for generation. Important implementation files are \texttt{src/models/laser.py}, \texttt{src/models/bottleneck.py}, \texttt{src/models/spatial_prior.py}, \texttt{src/data/vctk.py}, and \texttt{src/audio\_logging.py}. The paper source was written from the local run notes and artifacts under \texttt{/scratch/xl598/runs} and from the documented repo configs.

\section{Conclusion}

LASER replaces a single discrete latent code with a sparse dictionary representation and trains a transformer prior over the resulting support/coefficient tokens. The current vision experiments show that this representation can improve distributional reconstruction quality and produce tractable stage-2 sparse-token training. The audio extension is implemented and has a completed VQ-VAE reference, but a final LASER audio stage-2 experiment remains necessary before making strong cross-modal claims.
